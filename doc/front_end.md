# 项目说明

这是一个NanoKVM-Pro的测试软件，使用Tauri + React + Typescript + Vite开发。以下是设计细节：

请严格按照我提供的 Figma 链接内容生成 HTML 前端页面。UI 要严格还原设计稿，需要实现响应式设计进行开发：

[暗黑主题](https://www.figma.com/design/4FUSKWrZX46W6edi4cVAOi/NanoKVM-%E4%BA%A7%E6%B5%8B?node-id=9-126&t=jFMkBAt1NJaxCvYH-4);

### 前端页面文字说明：

页面分为左边栏和右边的内容区域。

左边栏从上到下包含以下内容：
+ 软件标题：NanoKVM-Pro产测工具（加粗）
+ 软件版本：V2.0
+ 本机编码：1（可以双击后修改为‘1’-‘F’单位16进制数字，如果是小写字母自动转换为大写）
+ 服务器状态：未连接/连接（红绿圆点表示连接状态）
+ 待上传数量：数字
+ 当前硬件：字符串
+ 当前序列号：九位字母数字组合
+ 目标IP：IPv4
+ 软件颜色选择：跟随系统/明亮/暗黑
+ 当前时间：年月日 - 时分

右侧内容区域显示测试的进程和项目，包含三种样式：
+ 大标题（加粗靠左显示），每个大标题之间使用细线隔离
+ 小标题
+ 状态框（包含文本+样式）以下几种样式用来表示此事的状态
    + 未开始：灰色
    + 正在测试：灰色+绿色边框（绿色边框围绕状态框转圈）
    + 正在修复：灰色+橙色边框（橙色边框围绕状态框转圈）
    + 测试完成：绿色
    + 测试失败：红色
所有内容使用json文件配置，json文件包含大标题、小标题、状态框（ID+文本+是否作为按钮+是否显示）

### 关于颜色
软件以不同的黑白灰为主体，具体颜色完全按照Figma设计稿完成，对于状态使用三种颜色表示：
+ 在明亮主题下：红（FFC9C9）、绿（B9F8CF）、橙（FEE685）
+ 在暗黑主题下：红（C10007）、绿（008236）、橙（BB4D00）

### 前后端交互

前端始终接收后端发来的状态并更新渲染画面
部分前端可以按下/输入的内容，在按下/输入后调用后端的API传递给后端

## 后端
使用rust构建，可以主动向前端发送状态，也可以接受来自前端的状态
预留以下api：
+ 接收本机编码
+ 发送服务器状态
+ 发送待上传数量
+ 发送当前硬件字符串
+ 发送当前序列号
+ 发送目标IP
+ 发送状态框状态（ID, 文本, 状态, 是否显示, 是否作为按钮）
+ 如果作为按钮需要有接收按下的函数
在main里创建两个进程
+ 进程1: 模拟产测过程
+ 进程2: 暂时delay+print
